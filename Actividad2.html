Introducción

¿Qué es BioMarina?

Aplicación Web Responsiva

Repositorio centralizado de información sobre especies marinas que funciona como un catálogo digital completo y accesible.

Objetivo Principal

Brindar información clara, organizada y accesible sobre biodiversidad de peces para apoyar la educación, el turismo sostenible y la toma de decisiones en pesca responsable.

Usuarios Objetivo:

Estudiantes Turistas Pescadores

Funcionalidades Principales

Fichas Técnicas Detalladas

Nombre científico, hábitat, estado de conservación

Buscadores con Filtros Avanzados

Filtrado por múltiples criterios

Descarga de Reportes Educativos

PDF y Excel/CSV

Panel Administrativo

Gestión de contenidos por expertos

Entregable MVP

Producto funcional entregado en 8-10 semanas con todas las funcionalidades core implementadas.

Cronograma

Planificación del MVP

Duración Total: 10 Semanas

1-2

Definición y Diseño

Semanas 1-2

  • Análisis de Requisitos y Benchmarking (FishBase)
  • Prototipado en Figma (wireframes alta fidelidad)
  • Modelado de Datos (base de datos georreferenciada)
  • Configuración del Entorno (Notion, servidor 99.5%)
Hitos: EDT, Prototipos

3-5

Desarrollo Core y API

Semanas 3-5

  • Desarrollo de API REST documentada
  • Implementación del Panel de Administración (CRUD)
  • Carga y Validación de Datos (50 especies, 98% precisión)
Hitos: API funcional, Panel Admin

6-8

Frontend y UX

Semanas 6-8

  • Motor de Búsqueda y Filtros Avanzados
  • Módulo de Exportación (PDF, Excel/CSV)
  • Optimización de Rendimiento (< 3 segundos)
  • Integración UI/UX (Figma a código)
Hitos: Web responsive, Exportación

9

QA y Ajustes

Semana 9

  • Pruebas de Estrés y Dispersión
  • Ciclo PDCA (Planear-Hacer-Verificar-Actuar)
  • Aplicación de Pareto (20% errores críticos)
Hitos: Corrección de bugs

10

Cierre y Entrega

Semana 10

  • Auditoría de Calidad Final
  • Entrega de Documentación Técnica
  • Firma del Acta de Aceptación del MVP
Hitos: MVP Entregado

Gestión del Alcance

WBS, Requisitos y PMBOK

Requisitos del Sistema

ID Requisito Prioridad Descripción
Requisitos Funcionales (RF)
RF-01 Plataforma Web Responsive ALTA Aplicación accesible y funcional en dispositivos móviles, tablets y escritorio.
RF-02 Ficha Técnica Completa ALTA Cada especie incluye: nombre común/científico, hábitat, distribución, tamaño, conservación, temporadas y características.
RF-03 Buscador con Filtros ALTA Filtrar especies por región, hábitat, tamaño, tipo y estado de conservación.
RF-04 Panel de Administración ALTA Interfaz segura para crear, editar, eliminar especies y cargar imágenes (JPG/PNG).
RF-05 API Documentada ALTA API REST disponible para consumo futuro por Proyectos #2 (Gastronomía) y #3 (Trazabilidad).
RF-06 Módulo de Exportación MEDIA Generar reportes en PDF (ficha individual) y Excel/CSV (listado completo).
Requisitos No Funcionales (RNF)
RNF-01 Rendimiento - Tiempo de carga máximo de 3 segundos por página en conexión estándar.
RNF-02 Calidad de Dato - La información biológica debe ser verificada por un experto antes de publicación.

Alcance - Exclusiones

  • App móvil nativa
  • Redes sociales integradas
  • Pagos en línea
  • Autenticación de usuarios públicos
  • Más de 50 especies en esta fase

* Se dejan para una Fase 2 del proyecto

Procesos del PMBOK Aplicados

01
Planificar Gestión del Alcance

Definir enfoque híbrido: PMBOK para gestión documental + Scrum para desarrollo ágil.
Notion para EDT y cambios, Figma para prototipos. (6 Resultados Esperados)

02
Recopilar Requisitos

Priorizar requisitos críticos mediante entrevistas con patrocinador y benchmarking.
Clave: Catálogo con 50 especies, API documentada, Web responsive.

03
Definir el Alcance

Incluye: App web, fichas, buscador, panel admin, BD, exportación.
Restricciones: Tiempo máximo 10 semanas, presupuesto limitado a recursos universitarios.

04
Crear EDT/WBS

Descomponer el alcance en entregables (no actividades). Nivel 1: Proyecto, Nivel 2: Fases (5), Nivel 3: Entregables (6 RF + 2 RNF), Nivel 4: Paquetes de trabajo.

05
Validar el Alcance

Proceso formal de aceptación de entregables completados por el Patrocinador (Docente). 6 Resultados Esperados. Evidencia: URL, capturas, revisiones, demos.

06
Controlar el Alcance

Supervisar estado y gestionar cambios. Prevenir scope creep y gold plating. Proceso formal de cambios. Congelamiento desde Semana 8. Actualización línea base.

Gestión de la Calidad

Plan y Métricas

Objetivo: Asegurar que los datos científicos de biodiversidad oceánica sean precisos, accesibles y útiles para la educación y pesca responsable.

Estrategia de Calidad

  • Calidad desde el Diseño: La revisión de calidad no ocurre al final, sino en cada iteración del software.
  • Prevención sobre Inspección: Implementar pruebas automatizadas y revisiones de código semanales para evitar el retrabajo.
"Entregar software que transforme datos oceánicos complejos en decisiones simples, garantizando integridad científica total y una disponibilidad del sistema del 99.5%."

Metodología de Control

  • Análisis de Valor Ganado (EVM): Comparar costo y tiempo real vs. planificado (tolerancia ±10%)
  • Demos Semanales: Validar alcance con los 6 Resultados Esperados
  • Checklists de Calidad: Verificación antes de marcar como "Done"

Validación: Criterios de Aceptación

01 App Web Responsive
Crítico

Criterio: Funcional en móvil, tablet y desktop (Chrome/Edge).

Evidencia: URL pública + Capturas en 3 dispositivos.

02 Ficha de Especie
Crítico

Criterio: Campos completos (hábitat, veda, conservación) + Imagen.

Evidencia: Revisión aleatoria de 10 fichas en producción.

03 Buscador y Filtros
Crítico

Criterio: Filtros (región, tamaño, tipo) retornan datos coherentes.

Evidencia: Grabación de prueba de filtros.

04 Panel Administrador
Crítico

Criterio: CRUD de especies y carga de imágenes operativo.

Evidencia: Demostración en vivo de creación de especie.

05 Base de Datos + API
Crítico

Criterio: API documentado (Swagger) y consumible por Proyectos #2 y #3.

Evidencia: Documentación técnica + Prueba de endpoint.

06 Exportación/Descarga
Importante

Criterio: Generación de PDF (ficha) y Listado (Excel/CSV) sin errores.

Evidencia: Archivos descargados y validados.
